Governance
==========

Committers are added by majority agreement of existing committers. To remain a
committer, one must continue to review, respond to bugs, keep up with the code.

Coding style
============

Readable code is important. PEP8 is preferred. Whatever works.

Testing
=======

All commits to trunk and other release branches should always pass ``make check``.

VCS
===

``bzr`` is the VCS for lmirror; trunk is branchable by doing
``bzr branch lp:lmirror``.
